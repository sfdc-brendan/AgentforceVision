Support that sees what you see

This is a self-contained local demo of Agentforce Vision. Connect it to your own Agentforce agent by pasting your Embedded Messaging code snippet in Settings - then open the chat, upload a photo of a device or error screen, and let the agent diagnose it and walk you through the fix.

View on GitHub
1

Paste your snippet

Open Settings and paste the Embedded Messaging code snippet from your Salesforce deployment. It's stored only in this browser (localStorage).

2

Open the chat

Salesforce's messaging widget loads in the corner and connects straight to your Agentforce agent - no credentials, no backend.

3

Upload a photo

Use the attachment button in the chat to send a photo. The agent analyzes it with the vision prompt template and grounds the fix in your Knowledge base.

Before you connect

  • In Setup → Messaging Settings, edit your channel and enable "Let customers send attachments to agents" so photo upload appears.
  • In Setup → Embedded Service Deployments, add this page's origin (for example http://localhost:8080) to the deployment's allowed/trusted origins, or the widget won't load.
  • The snippet looks like the one Salesforce generates for your Messaging for Web deployment - it contains embeddedservice_bootstrap.init(...) and a bootstrap.min.js script tag.
